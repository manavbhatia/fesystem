void FESystemUtility::TimeLogs::setEvent ( const std::string &  event_name,
const std::string &  header 
)

this method sets the event under the given header. This behaves in the following way:

-- if no other event is being logged in the given header, then this event defines the first event and is started

-- if the current event in the header being logged is the same as the given event, then it checks if the event was paused or not. If it was paused, it restarts the event. Otherwise, it adds to the call counter of the event

-- if the current event in the header is different, then if the event is not already paused, it pauses the event, and sets the current logged event to the given event
