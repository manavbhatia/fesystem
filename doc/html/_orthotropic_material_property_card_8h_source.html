00027 /// The \p PropertyCard class that provides the necessary data structure 
00028 /// for storage of material and structural properties. This is a base class, 
00029 /// and different material cards can derive from this. Each property card
00030 /// can have dependence on some parameter values, which is defined through the input. If a 
00031 /// parameter dependence has been defined, then before the parameter values can be accessed, 
00032 /// the card needs to be initialized at some parameter values. During reinitialization, if
00033 /// no value has been specified for a parameter, a default value for the same is assumed.
00034 /// Between any two consecutive initializations, the card needs to be cleared of any previous
00035 /// initialization. This is needed to make sure that the card is cleared of any remnant value
00036 /// initializations to avoid errors. Once initialized, the value of the property can be accessed from
00037 /// the card.
00038 //
